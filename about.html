Project Description

Data Source: PG Open Data


Information Problem:

The information problem is that the residents of College Park/Prince George’s County may not know about polling places where they can practice their civic right to vote. This could be due to the information not being readily available or difficulty when navigating through the information. Anyone that is eligible to vote should be able to have easy access to information regarding their polling location. There are many groups of stakeholders that care about this problem. One primary stakeholder is the population of College Park/Prince George’s County. As UMD is within Prince George’s County, a substantial portion of the commuter population may be from within the county and is relevant to them. This also includes UMD students of voting age that have changed their permanent address to one that is within Prince George’s County. Our system would be able to solve this information problem because it will provide an easy way to navigate polling location data along with potentially adding new information


Project:

Using HTML,CSS, and JS. We used leaflet along with a couple plugins such as AJAX to extract the GeoJSON data and make an aesthetically pleasing map that will allow users to find polling stations in PG County. We loaded external PG county polygons from GeoJson data to distinguish the county from the rest of the map. We then added the external PG County polling place GeoJson data and added the coordinates as markers on the map. We styled them with an voting box icon styling the markers and added relevant information in the popups such as the name, the address, and a link that directs the user to google maps with the address already entered. We then added a functional full screen control button in which the user could toggle on and off. Lastly, we made a list of all 274 current locations. This project serves as a basic solution for having a map and list of all current locations. The map can simply be updated by loading in a GeoJson file with new data. This project acts as a small contribution to the problem. In the future, a sidebar with the full information from the properties of each marker (ie. District, School District, Legislative District, etc.) could be added with event listeners that update the sidebar when a new marker is clicked on. Additionally, a control with filters could be added that can filter the polygon data, polling stations by city, district, etc. Lastly, in the future we would hope to add a plug in that allows the user to enter their address and return the closest polling station to them.
